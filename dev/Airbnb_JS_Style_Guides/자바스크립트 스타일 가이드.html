# 이터레이터 사용: 고차 함수 > 루프 for-in / for-of

  • 고차 함수
    • 불변 규칙을 강제하기 위함이다.
    • 값을 리턴하는 순수 함수를 다루는 것이 더 쉽기 때문에 사이드 이펙트도 작을 수 있음.
  • 배열 순회
    • map()
    • every()
    • filter()
    • find()
    • findIndex()
    • reduce()
    • some() / ...
  • 객체 -> 배열
    • Object.keys()
    • Object.values()
    • Object.entries()
const numbers = [1, 2, 3, 4, 5];

// bad
let sum = 0;
for (let num of numbers) {
  sum += num;
}
sum === 15;

// good
let sum = 0;
numbers.forEach((num) => {
  sum += num;
});
sum === 15;

// best (use the functional force)
const sum = numbers.reduce((total, num) => total + num, 0);
sum === 15;

// bad
const increasedByOne = [];
for (let i = 0; i < numbers.length; i++) {
  increasedByOne.push(numbers[i] + 1);
}

// good
const increasedByOne = [];
numbers.forEach((num) => {
  increasedByOne.push(num + 1);
});

// best (keeping it functional)
const increasedByOne = numbers.map((num) => num + 1);

# 단항 증감 (++, --) X

  • 단항 증감은 세미콜론 자동 삽입의 대상이 된다.
    • silent errors 의 원인이 될 수 있다.
  • num++ / num ++ 보다 num += 1 이 더 바람직하다.
  • 단항 증감을 사용하지 않으면,
    • pre-incrementing/pre-decrementing 실수를 방지할 수 있다.
// bad

const array = [1, 2, 3];
let num = 1;
num++;
--num;

let sum = 0;
let truthyCount = 0;
for (let i = 0; i < array.length; i++) {
  let value = array[i];
  sum += value;
  if (value) {
    truthyCount++;
  }
}

// good

const array = [1, 2, 3];
let num = 1;
num += 1;
num -= 1;

const sum = array.reduce((a, b) => a + b, 0);
const truthyCount = array.filter(Boolean).length;
